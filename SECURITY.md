# Security Policy

## Supported versions

보안 수정은 최신 GitHub Release와 `main` 브랜치를 기준으로 제공합니다. 가능하면 항상 최신 버전을 사용해 주세요.

## Reporting a vulnerability

보안 문제는 공개 Issue에 작성하지 말고 GitHub의 [private vulnerability reporting](https://github.com/Kyuhan1230/ai-usage-monitor/security/advisories/new)을 이용해 주세요.

신고 내용에는 가능한 범위에서 다음 정보를 포함해 주세요.

- 영향을 받는 버전과 Windows 버전
- 재현 단계와 예상 영향
- 최소한의 증명 자료
- 제안하는 완화 방법이 있다면 해당 내용

인증 토큰, 브라우저 쿠키, 원본 Codex/Claude 세션 JSONL 또는 개인 식별 정보는 보내지 마세요. 재현에 로그가 필요하면 먼저 민감 정보를 제거하고 필요한 최소 부분만 공유해 주세요.

유지관리자는 신고를 확인한 뒤 재현 가능 여부와 대응 방향을 회신합니다. 수정이 준비되기 전에는 공개를 미뤄 주시길 요청드립니다.
