# v0.3.0 단계 기록: 로컬 Usage Insights

작성일: 2026-07-18

## 이번 단계에서 한 일

- 날짜별 한도 히스토리에서 현재 reset 주기를 분리하고 시간당 소진율과 예상 고갈 시각을 계산했습니다.
- 잔여 25%를 주의, 10%를 위험으로 판정하고 현재 속도로 reset 전 고갈되는 경우도 경고합니다.
- 최근 구간 속도를 이전 구간 중앙값과 MAD에 비교하고, 날짜별 토큰 누계를 최근 7일 기준과 비교해 이상 급증을 감지합니다.
- 오늘/전일과 최근 7일/이전 7일 토큰 누계를 비교합니다.
- 모델별 입력·캐시·출력 토큰을 공식 표준 API 정가로 환산하고, 같은 토큰을 저비용 모델에 적용했을 때의 이론적 절약액을 계산합니다.
- 결과를 별도 Usage Insights 창과 Windows 로컬 알림으로 표시하고, 필요한 감속 비율과 점검할 행동을 결정론적 규칙으로 제안합니다.

## 가격 추정 원칙

- OpenAI와 Anthropic 공식 가격 문서에서 2026-07-18에 확인한 1M 토큰당 표준 API 정가를 코드에 기준일과 함께 기록했습니다.
- 이 값은 구독 청구액이 아니라 API 정가 등가 추정입니다.
- 가격을 모르는 모델의 토큰은 비용 합계에서 제외하고 가격 매칭 비율을 함께 표시합니다.
- 단순 작업의 모델 전환 가능성만 보여주며, 작업 품질이 같다고 단정하지 않습니다.

## 프라이버시

- 원본 JSONL은 로컬에서만 읽습니다.
- 캐시와 분석 파일에는 날짜, 제공자, 모델, 토큰 합계, 한도 비율과 계산 결과만 저장합니다.
- Claude statusLine 입력과 `/usage` 출력 원문은 상태 파일과 히스토리에 저장하지 않습니다.
- 네트워크 텔레메트리나 원격 분석 서버는 추가하지 않았습니다.

## 검증 기준

- 합성 시계열에서 시간당 소진율, reset 전 고갈 판정, 임계치와 이상 급증이 예상값과 일치해야 합니다.
- 합성 Codex/Claude JSONL에서 모델 변경과 Claude 메시지 ID 중복 제거가 동작해야 합니다.
- 공식 가격표를 적용한 입력·캐시·출력 비용과 모델 절약액이 예상값과 일치해야 합니다.
- 모든 renderer와 수집기 구문 검사, 기존 통합 테스트, 문서 스크린샷 렌더, `git diff --check`가 통과해야 합니다.

## 다음 단계

- v0.4.0에서 Python/FastAPI 전체 대시보드와 legacy poller·PowerShell 실행 경로를 제거하고 Electron 앱 안의 단일 로컬 화면으로 통합합니다.
