# 1.0.3 실행 중 PATH 갱신

## 문제

Codex CLI 설치 프로그램은 사용자 PATH에 설치 디렉터리를 추가하지만 이미 실행 중인 Windows 프로세스의 환경 블록은 바뀌지 않습니다. 트레이에서 대기하던 모니터는 설치 전 PATH로 `where codex`를 실행해 새 CLI를 찾지 못했습니다.

## 수정

- 명령 탐색 시 프로세스 PATH와 함께 레지스트리의 현재 사용자·시스템 PATH를 읽습니다.
- PATH의 각 디렉터리에서 `.exe`, `.cmd`, `.bat` 실행 후보를 직접 확인합니다.
- 보호된 Codex 데스크톱 번들을 제외하는 기존 검사는 그대로 적용합니다.
- Setup을 열거나 상태를 확인할 때 CLI를 실행하지 않으며 파일 존재 여부만 확인합니다.

이제 실행 중 CLI를 설치한 뒤 **지금 수집**을 누르면 앱 재시작 없이 새 경로를 사용합니다.
