# 1.0.5 첫 실행 설치·인증 온보딩

## 문제

1.0.4 설치 프로그램은 Codex CLI 누락만 보완했다. Claude Code 설치 여부는 확인하지 않았고, Setup은 두 명령의 존재만 확인한 뒤 과거 사용량 수집 성공 여부를 로그인 상태처럼 표시했다. 사용자는 `설치됨`, `로그인됨`, `사용량이 수집됨`을 구분하기 어려웠다.

## 결정

- 설치 프로그램은 Codex CLI와 Claude Code의 공식 Windows 설치 위치를 각각 확인한다.
- 없는 도구만 기본값이 **아니요**인 개별 동의 대화상자를 표시한다.
- 첫 일반 실행은 Compact 대신 Setup을 열며 Windows 자동 실행의 `--background` 모드는 계속 WebView를 만들지 않는다.
- Setup 진입 시 Codex는 `codex login status`, Claude는 `claude auth status`를 최대 8초 동안 병렬 실행한다.
- 인증 명령 출력은 저장·표시·로그 기록하지 않고 종료 성공 여부만 `authenticated / unauthenticated`로 변환한다.
- 상태를 `미설치 / 로그인 필요 / 로그인 완료 / 확인 실패`로 구분하고 필요한 설치·로그인 버튼만 활성화한다.
- Setup의 설치 버튼도 사용자 확인 후 공식 설치 스크립트를 보이는 PowerShell 창에서 실행한다.
- 두 로그인이 완료되면 **설정 완료**를 활성화한다. 사용자는 **나중에**를 선택할 수도 있다.
- 인증 상태 확인과 사용량 수집을 분리한다. 첫 Setup 진입과 **상태 다시 확인**은 인증 상태만 확인하며, **사용량 확인**만 기존 원샷 수집을 실행한다.

## 개인정보 경계

- 인증 상태는 디스크에 저장하지 않는다.
- CLI stdout/stderr는 Rust 프로세스 메모리에서 성공 여부를 판정한 즉시 버린다.
- `onboarding.json`에는 완료·건너뜀 여부와 시각만 저장한다.
- 설치, 로그인, 사용량 수집은 사용자 동작 없이 실행하지 않는다.

## 검증 기준

1. 양쪽 CLI가 없으면 설치 프로그램이 두 번의 독립적인 선택을 제공한다.
2. 어느 한쪽만 있으면 없는 도구만 묻는다.
3. 무인 설치는 질문과 다운로드가 없다.
4. 첫 일반 실행은 Setup을 열고 백그라운드 자동 실행은 창이나 CLI를 시작하지 않는다.
5. 설치된 두 CLI의 인증 상태가 직접 명령 결과와 일치한다.
6. 초기 Setup 확인은 사용량 app-server 또는 `/usage`를 실행하지 않는다.
7. 두 인증이 완료되어야 **설정 완료**가 활성화된다.
