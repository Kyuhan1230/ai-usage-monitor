# v1.0.1 — Windows 보조 창 흰 화면 수정

## 증상

Compact 창에서 Setup, Usage Insights 또는 Token Details를 열면 네이티브 창만 생성되고 WebView 내용은 흰 화면으로 남을 수 있었다.

## 원인

Windows WebView2 창을 동기 Tauri command 및 tray event handler 안에서 생성했다. Tauri가 문서화한 Windows 교착 조건과 일치한다.

## 변경

- 프런트엔드 command에서 시작되는 창 생성은 async command와 `spawn_blocking`을 사용한다.
- tray menu와 tray click에서 시작되는 창 생성은 별도 worker thread로 보낸다.
- 앱 시작 시 Compact 창을 만드는 setup hook은 기존대로 유지한다.
- UI 정적 테스트가 동기 보조 창 생성의 재도입을 막는다.

## 검증 기준

- Rust release 컴파일과 테스트가 통과한다.
- Compact에서 Setup을 열었을 때 HTML/CSS 내용이 표시된다.
- Setup을 닫은 뒤 다시 열어도 흰 화면이 발생하지 않는다.
- tray 메뉴에서도 Setup, Usage Insights, Token Details가 정상 표시된다.
