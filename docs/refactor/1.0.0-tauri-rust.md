# v1.0.0 — Tauri/Rust 경량 로컬 앱

> **v1.2.0 정책 변경:** v1.0.0에서 제거했던 자동 업데이트 확인은 [자동 업데이트 구현 인수인계](../UPDATER_IMPLEMENTATION_HANDOFF.md)에 따라 하루 최대 한 번 확인하고 사용자 승인 뒤에만 설치하는 Tauri updater로 다시 도입했다. 아래 내용은 v1.0.0 당시의 설계 기록이다.

## 목표

v0.4.0의 기능과 로컬 전용 경계를 유지하면서 Electron/Chromium 번들 비용을 제거하고, 앱 시작·대기 중 숨은 CLI와 listening port가 전혀 없는 Windows 트레이 앱으로 전환한다.

## 변경

- Electron/Node 백엔드와 패키징 도구를 Tauri 2/Rust로 교체했다.
- 원샷 계정 수집, Claude statusLine 훅, JSONL 증분 집계, 예측·알림·비교·비용·추천 계산을 Rust로 이식했다.
- Compact, Usage Insights, Token Details, Setup 화면은 프레임워크 없는 기존 HTML/CSS/JS를 유지하고 Tauri command bridge만 사용한다.
- 로그인 시작은 `--background` 트레이 모드로 등록한다. 이 모드는 WebView와 사용량 CLI를 만들지 않는다.
- 사용자가 트레이 또는 메뉴에서 요청할 때만 WebView를 만들고, 창을 닫으면 WebView를 파기한다.
- Codex/Claude CLI 수집은 사용자의 새로고침 동작에서만 단발 실행한다.
- 자동 업데이트 확인을 제거했다. 새 버전 설치는 사용자가 GitHub Release에서 명시적으로 시작한다.

## 기능 불변 조건

- Codex 5-hour/weekly 및 Claude session/weekly 한도를 표시한다.
- 한도 히스토리에서 소진 속도, 예상 고갈 시각, reset 전 고갈 여부와 신뢰도를 계산한다.
- 잔여 25%/10% 임계치, 토큰 이상 급증, 전일·전주 대비를 계산한다.
- API 정가 등가 비용, 저비용 모델 절약 가능성, 규칙 기반 실행 추천을 제공한다.
- 날짜·모델별 토큰 상세는 최대 500행으로 앱 안에서만 표시한다.
- 인증 토큰, 쿠키, 프롬프트와 응답 본문을 분석 파일에 복사하지 않는다.
- HTTP 서버, 원격 텔레메트리, 자동 업데이트 확인과 listening port를 만들지 않는다.

## 테스트 게이트

- Rust 단위 테스트: 7개 통과(수집 파싱, 원문 제거, 저장 중복 제거, JSONL 캐시, 분석 지표와 가격 규칙).
- UI/구성 회귀 테스트: 6개 JavaScript 구문, bridge 순서, 버전 일치, 서버 부재, 시작 시 자동 수집 부재, 지연 WebView 구성을 검증했다.
- `cargo fmt --check`, `cargo clippy --all-targets -- -D warnings`와 Tauri release/NSIS build를 통과했다.
- CI와 Release workflow는 NSIS 설치 파일이 20MB를 넘으면 실패한다.

메모리 수치는 단일 PC의 working set 참고값이다. 설치 크기와 대기 자원은 크게 줄었지만 UI가 열린 동안의 WebView2 비용은 Electron 기준선과 거의 같다. 따라서 v1은 이를 숨기지 않고, UI를 닫으면 WebView를 완전히 파기하는 구조로 해결한다.

## 배포 경계

- 설치본은 Rust 애플리케이션과 정적 UI만 포함한다. Electron, Node.js, CPython과 Chromium 런타임을 번들하지 않는다.
- 렌더링은 Windows에 설치된 Microsoft Edge WebView2 Runtime을 사용한다.
- NSIS의 WebView2 설치 모드는 `skip`으로 고정해 설치 중 자동 다운로드도 하지 않는다.
- 현재 공개 설치 파일은 SignPath Foundation 승인 전까지 Authenticode 미서명일 수 있다.

- 모델 가격표를 릴리스마다 공식 출처와 대조하는 체크리스트 자동화.
