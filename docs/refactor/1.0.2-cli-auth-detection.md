# 1.0.2 CLI 인증 경로 수정

## 관찰된 문제

- `where codex`가 Microsoft Store Codex 데스크톱 패키지의 `app\\resources\\codex.exe`를 반환했지만, 외부 데스크톱 앱이 이 파일을 실행하면 Windows가 접근을 거부했습니다.
- Claude Code 2.1.183에서 `claude auth`는 인증 도움말만 출력하며 실제 로그인은 `claude auth login`입니다.
- Setup은 자식 터미널이 열린 뒤 명령이 실패해도 사용자에게 성공처럼 보였고 해결 행동을 제시하지 않았습니다.

## 변경한 경계

1. `Program Files\\WindowsApps\\OpenAI.Codex_*\\app\\resources\\codex*`는 데스크톱 전용 번들로 분류하고 CLI 후보에서 제외합니다.
2. 로그인과 수집은 PATH의 이름을 다시 해석하지 않고 선택된 실제 경로를 사용합니다.
3. Setup 스냅샷은 CLI를 실행하지 않고 `ready`, `desktop_bundle_only`, `missing` 상태만 반환합니다.
4. 로그인 가능 상태에서는 `codex login` 또는 `claude auth login`을 열고, 불가능 상태에서는 명시적 사용자 클릭으로 공식 설치 문서만 엽니다.

## 프라이버시 영향

- Setup을 열거나 상태를 표시하는 것만으로 Codex·Claude 프로세스나 네트워크 요청을 시작하지 않습니다.
- 공식 설치 문서는 사용자가 **CLI 설치 안내**를 직접 눌렀을 때 기본 브라우저에서만 열립니다.
- 인증 정보와 로컬 경로는 renderer에 전달하거나 저장하지 않습니다.
