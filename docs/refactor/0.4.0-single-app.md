# v0.4.0 — single app, no local server

## 목표

동일한 최신 상태·분석 결과를 유지하면서 Python/FastAPI 대시보드와 legacy 실행 경로를 제거한다.

## 변경

- 모델·날짜별 토큰 상세를 `details.html` Electron 화면으로 이전했다.
- Python 소스, requirements, 번들 런타임 준비 스크립트와 서버 시작 코드를 삭제했다.
- node-pty와 과거 PowerShell wrapper/tray 스크립트를 삭제했다.
- snapshot 및 IPC 이름을 `details`로 통일했다.
- 현재 원샷 수집, statusLine 이벤트, 증분 토큰 집계와 인사이트만 검증하도록 테스트를 재작성했다.

## 불변 조건

- Codex app-server는 요청 시 한 번만 실행되고 종료된다.
- Claude statusLine이 기본이며 `/usage`는 시작·수동 새로고침의 대체 경로다.
- 분석은 소진 예측, 임계치, 비교, 이상 급증, 비용, 모델 절약과 추천을 계속 제공한다.
- 프롬프트·응답 본문과 인증 정보는 저장하지 않는다.

## 검증 게이트

- JavaScript 회귀·통합 테스트
- 네 renderer 기반 README 스크린샷 생성 및 시각 확인
- 소스·workflow·패키지의 Python, FastAPI, node-pty, listening server 부재 검사
- Electron unpacked package 생성

실측 결과:

- NSIS 설치 파일: 95.4 MB
- 애플리케이션 ASAR: 2.3 MB
- 패키지 실행 6초 후: Electron 프로세스 4개, 합산 working set 425.8 MB
- 앱 프로세스의 listening port: 0개

서버와 Python 제거는 성공했지만 Chromium 런타임 비용은 그대로 남았다. 이 수치를 v1.0.0 경량화 기준선으로 사용한다.
