# v0.2.0 단계 기록: 이벤트·원샷 수집

작성일: 2026-07-18

## 이번 단계에서 한 일

- Codex 수집을 숨은 대화 세션의 `/status` 폴링에서 공식 app-server의 `account/rateLimits/read`와 `account/usage/read` 원샷 호출로 교체했습니다.
- app-server는 `initialize` → `initialized` 순서를 지키고 계정 메서드 응답을 받은 즉시 종료합니다.
- Claude의 기본 수집 경로에서 1분 주기 `/usage` 프로세스를 제거했습니다.
- Claude는 statusLine 이벤트로 자연스럽게 갱신하고, 앱 시작과 사용자의 수동 새로고침 때만 `/usage`를 한 번 호출합니다.
- 동시에 들어온 새로고침 요청은 하나의 작업으로 합쳐 중복 CLI 실행을 막습니다.

## 유지한 것

- 기존 상태 JSON 스키마와 컴팩트 화면의 표시 형식은 유지했습니다.
- 구버전 사용자를 위한 독립 실행형 legacy poller 파일은 이번 단계에서 삭제하지 않았습니다. 기본 앱 경로에서는 더 이상 참조하지 않으며 v0.4.0에서 제거합니다.
- Python/FastAPI 전체 대시보드는 이번 단계에 남겨 두었고, 버튼을 눌렀을 때만 실행됩니다.

## 검증 기준

- 메인 Electron 파일에 poller 시작·재시작·PID 감시 경로가 없어야 합니다.
- Codex 모의 app-server가 `initialize`, `initialized`, `account/rateLimits/read`, `account/usage/read`만 받아야 합니다.
- 원샷 캡처가 끝나면 상태 JSON과 날짜별 history JSONL이 각각 기록되어야 합니다.
- 전체 테스트와 `git diff --check`가 통과해야 합니다.

## 다음 단계

- v0.3.0에서 로컬 history를 바탕으로 소진 속도, 고갈 예상 시각, 임계치, 비교, 이상 급증, 비용 및 실행 가능한 추천을 계산하고 화면에 표시합니다.
