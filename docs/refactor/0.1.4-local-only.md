# v0.1.4 단계 기록: localhost 전용 대시보드

생성일시: 2026-07-18 00:43:19 +09:00
수정일시: 2026-07-18 00:47:00 +09:00

## 이번 단계에서 한 일

- 모든 대시보드 실행 경로를 `127.0.0.1`에만 바인딩했습니다.
- package 버전을 `0.1.4`로 올리고 변경 기록을 추가했습니다.
- 외부 인터페이스 바인딩 회귀를 막는 테스트를 추가했습니다.

## 지킨 것

- 상태 수집, 세션 집계, Electron UI와 updater 동작은 변경하지 않았습니다.
- 사용자가 명시한 로컬 우선 프라이버시 약속을 실행 경로 전체에 적용했습니다.

## 부족한 점과 남은 리스크

- Electron과 bundled Python 구조는 여전히 무겁습니다.
- Codex와 Claude 상태 수집은 아직 백그라운드 CLI 실행에 의존합니다.

## 다음 단계

- v0.2.0에서 Codex app-server와 Claude statusLine 기반 이벤트 수집으로 교체합니다.

## 테스트 결과

- Node.js 24.14.0과 Python 3.11 가상환경에서 전체 통합 테스트가 `all tests passed`로 완료됐습니다.
- legacy Python 서버가 실제로 `127.0.0.1:8799`에만 listen하는 것을 소켓 수준에서 확인했습니다.
- `git diff --check`를 통과했습니다.

## 출력 불변성 검증 결과

- 기존 통합 테스트로 HTML 렌더링과 status JSON 응답 형식이 유지됨을 확인했습니다.
- 네트워크 listen 주소만 외부 인터페이스에서 loopback으로 제한됐습니다.
